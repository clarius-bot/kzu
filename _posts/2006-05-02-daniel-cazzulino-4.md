## Saturday, May 15, 2004 - Posts

#####  [Mvp.Xml project: packed XML cool utilities](http://clariusconsulting.net/blogs/kzu/archive/2004/05/15/233.aspx)

I've released the ~~seventh~~ eighth "version" of the [ Mvp.Xml project](http://sf.net/projects/mvp-xml) on [ SourceForge](http://prdownloads.sourceforge.net/mvp-xml/Mvp.Xml.zip?download). This is a maintenance release with some bugs fixed in the [ XPathNavigatorReader](http://weblogs.asp.net/cazzu/archive/2004/04/19/115966.aspx). Thanks to Tom Smalley and Joost Ploegmakers!

So far, the package contains:

  * [XPathCache and DynamicContext](http://weblogs.asp.net/cazzu/archive/2004/04/02/106667.aspx)
  * [IndexingXPathNavigator](http://www.tkachenko.com/blog/archives/000194.html)
  * [XPathNavigatorReader](http://weblogs.asp.net/cazzu/archive/2004/04/19/115966.aspx)
  * [XmlNodeNavigator](http://www.tkachenko.com/blog/archives/000117.html)
  * [SubtreeXPathNavigator](http://weblogs.asp.net/cazzu/archive/2004/06/24/164243.aspx)
  * [XmlFragmentStream](http://weblogs.asp.net/cazzu/archive/2004/04/23/119263.aspx)
  * [XPathIteratorReader](http://weblogs.asp.net/cazzu/archive/2004/04/26/120684.aspx)
  * [XmlFirstUpperReader and XmlFirstLowerWriter](http://weblogs.asp.net/cazzu/archive/2004/05/10/129106.aspx)
  * [XmlNodeFactory](http://clariusconsulting.net/cazzu/archive/2004/05/31/144922.aspx) (to return well-formed XML from WebServices without XmlDocument)
  * [XGen for .NET v1.x](http://weblogs.asp.net/cazzu/posts/XGenToolRelease.aspx) (Strongly-typed, event-rising, design-time generated custom XmlSerializers to avoid first-time use performance impact on XmlSerializer plus another BIG bonus). 

Each feature is explained in a corresponding weblog post. 

**Update** : the [Mvp.Xml project](http://sf.net/projects/mvp-xml) now also powers the [Atom.NET](http://atomnet.sourceforge.net/) project, as explained in the [announcement post](http://loluyede.blogspot.com/2004/07/atomnet-grows-with-mvp-xml.html).

Enjoy!
